# 我的 Hexo 博客搭建之旅
> 记录在安卓 Termux 环境下搭建 Hexo 博客的踩坑与收获

## 一、 缘起：为什么选择 Hexo？
作为一个喜欢折腾的爱好者，我一直想拥有一个属于自己的个人博客。之前用过 WordPress、Typecho、Zblog后，最终选择了 **Hexo** —— 一款轻量级的静态博客框架，无需服务器，直接部署到 GitHub Pages 即可免费上线(主要是不需要额外购买服务器🌚)，对新手~~非常友好~~。

更重要的是，Hexo 支持在手机的 Termux 终端中操作，随时随地都能写文章、改配置，满足日常记录，也不需要电脑或者用远程连接。

## 二、 踩坑之路：那些遇到的问题
搭建过程并非一帆风顺，印象最深的是两个问题：

1.  **Termux 文件不可见**
    一开始在手机文件管理器里找不到 Hexo 的博客文件，后来才知道需要执行 `termux-setup-storage` 命令挂载存储，再通过 `~/storage/shared` 目录实现文件互通。

2.  **GitHub 连接超时**
    因为一些原因~~(懂得都懂)~~部署时频繁遇到超时、连接被拒绝，最终通过修改配置，以及切换 GitHub 镜像源解决了这个问题。

## 三、 博客美化：从简陋到个性化
Hexo 的魅力在于丰富的主题、插件生态还有不用买服务器:dog:，顺便整了点改动：
- **更换主题**：我用的是 Volantis 主题，感觉还是比较好看的。
- **添加标签**：写了一些标签，让文章分类一目了然了，有强迫症了属于是。
- **配置评论插件**：整上了评论插件，方便指出文章的不足。

## 四、 高效技巧：Termux 快捷指令
为了省事儿，我整了几个实用的快捷指令：
```bash
# Hexo 快捷指令
alias hc='hexo clean'
alias hs='hexo server -p 4000'
alias hgd='hexo clean && hexo g -d'
```

只用输入 `hgd` 就能一键清理缓存、生成静态文件并部署到github，雀食🐮🍺。

## 五、写在最后
从 0 到 1 搭建博客的过程，虽然踩了不少坑，但也收获了很多知识(然鹅并没有太多)。看着自己的博客从简陋的默认页面，慢慢变成喜欢的样子，那种成就感是无法言喻的。
 
如果你也想搭建自己的 Hexo 博客，别害怕遇到问题 —— 每一个坑都是一次成长的机会。
